Why use init scheme?

Init scheme can help you automatically set properties, columns and fields when transform model.

For example, the database has a book table, we need to transform a model. Often we need to add some properties and columns to the model.

There is a table in the database

Transform by default model, Can't meet our needs.

How to use init scheme

For example, according to business requirements, we need to add and modify the model.

  • Modify
    • type column: field type, such as String, Integer.
    • fullType column: field full type, such as java.lang.String, java.lang.Integer, java.util.Date.
  • Add
    • title property: title.
    • fieldName column: UI field name.
    • controlType column: UI control type, such as text, select and date.
    • isQuery column: Whether field to be queried.

Init Column

There are 2 kinds:

  • Existing column init
  • New column init
Existing column init

Including name, type, fullType:

  • name: Field name, which corresponds to the field name of the class.
  • type: Field type, default is null, can be modified. such as Stirng, int, Integer.
  • fullType: Field full type, default is null, such as java.lang.String、java.util.Date.

There are also two ways: default and convert.

The default value is filled with a fixed value. Fill in here myName.

The same value is generated for each field in the same column.

But in most cases, name, type, fullType these three field are converted by other columns

Our conversion scheme:

  • name: id --> converter headLowerCamel, the value of the id is the same as the field of the table, which is converted into a camel case, such as regist_time to registTime.
  • type: jdbcType --> Mapping type, jdbcType is the type of jdbc standard, we mapping to the type by jdbcType.
  • fullType: jdbcType --> Mapping fullType, we also mapping to the type by jdbcType.

Converter headLowerCamel, lower case and camel case.

Mapping type, from jdbcType to the development language type.

Mapping full type, from jdbcType to the development language fullType.
